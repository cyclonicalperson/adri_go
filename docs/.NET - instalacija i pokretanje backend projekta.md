# .NET instalacija i pokretanje backend projekta

## 1. Uvod

.NET je razvojna platforma koju je razvila kompanija Microsoft za izradu različitih vrsta aplikacija kao što su web aplikacije, desktop aplikacije, mobilne aplikacije i API servisi.

Za razvoj backend aplikacija najčešće se koristi ASP.NET Core koji omogućava izradu REST API servisa.

Prednosti .NET platforme:
- Visoke performanse
- Podrška za više operativnih sistema (Windows, Linux, macOS)
- Velika zajednica i dokumentacija
- Jednostavna integracija sa bazama podataka

---

## 2. Instalacija potrebnih alata

Za razvoj .NET backend aplikacije potrebno je instalirati sledeće alate:

### 2.1 Instalacija .NET SDK

.NET SDK omogućava kreiranje, build i pokretanje .NET aplikacija.

Koraci instalacije:

1. Otvoriti zvanični sajt:
   
   https://dotnet.microsoft.com/download

2. Preuzeti najnoviju verziju **.NET SDK**.

3. Pokrenuti instalacioni program i pratiti korake instalacije.

---

### 2.2 Instalacija Visual Studio okruženja

Visual Studio je razvojno okruženje koje olakšava razvoj .NET aplikacija.

Koraci:

1. Otvoriti sajt:

   https://visualstudio.microsoft.com/

2. Preuzeti **Visual Studio Community** verziju.

3. Tokom instalacije označiti opciju:

   - ASP.NET and web development

4. Završiti instalaciju.

---

## 3. Provera instalacije

Nakon instalacije potrebno je proveriti da li je .NET uspešno instaliran.

Otvoriti terminal ili Command Prompt i pokrenuti komandu: dotnet --version


Ako je instalacija uspešna, prikazaće se verzija instaliranog .NET SDK-a.

Primer: 8.0.100


---

## 4. Kreiranje novog backend projekta

Novi backend projekat može se kreirati pomoću .NET CLI alata.

Koraci:

1. Otvoriti terminal.
2. Preći u folder gde želimo da napravimo projekat.
3. Pokrenuti komandu: dotnet new webapi -n BackendProjekat

Ova komanda kreira novi ASP.NET Core Web API projekat.

Zatim ući u folder projekta: cd BackendProjekat


---

## 5. Pokretanje backend aplikacije

Backend aplikacija se pokreće sledećom komandom: dotnet run

Nakon pokretanja aplikacija će biti dostupna na adresi: https://localhost:5001 ili http://localhost:5000

Ako je uključen Swagger, API dokumentacija će biti dostupna na: https://localhost:5001/swagger


---

## 6. Osnovna struktura projekta

Primer strukture ASP.NET Core backend projekta:
BackendProjekat/
│
├── Controllers/
│ └── WeatherForecastController.cs
│
├── Program.cs
├── appsettings.json
├── BackendProjekat.csproj


Opis fajlova:

- **Controllers/** – sadrži API kontrolere
- **Program.cs** – glavna konfiguracija aplikacije
- **appsettings.json** – konfiguracija aplikacije
- **.csproj** – definicija projekta i zavisnosti

---

## 7. Zaključak

.NET platforma omogućava brz i efikasan razvoj backend aplikacija.  
Korišćenjem .NET SDK-a i ASP.NET Core frameworka moguće je jednostavno napraviti i pokrenuti REST API koji se može koristiti kao backend za web ili mobilne aplikacije.
